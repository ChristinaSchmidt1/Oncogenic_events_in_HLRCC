A y value of 10 would mean a rate of 1 error in 10, then a value of 20 would mean a rate of 1 error in 100 and a value of 30, 1 error in 1000. Per Base Sequence Content

The per base sequence content plot shows the proportion of each of the four DNA nucleotides at each position across the reads. In a random library we expect little or no difference between the proportions of the four nucleotides across the read's length, which means that the plot should show parallel lines across the x-axis. Some problems that can be detected using these plots are over-represented sequences, biased fragmentation and biased composition libraries. Some RNA-Seq library preparation methods will cause a bias at the beginning of the reads, as can be seen here. It is also common to see a drop in the proportion of A nucleotide at the end of the read due to trimming of adapters.
The plot looks as we would expect it for this type of data.



Per Sequence GC content

The next plot shows the distribution of GC content in a set of reads. Non-biased libraries should show a normal distribution of GC content, with distribution centered at the overall GC content of the underlying genome. This plot can help to identify contamintation or a biased subset in the library.
The plot doesn't show any evidence of contamination and the peaks are well aligned.

Sequence duplication levels

The sequence duplication levels plot shows if there are identical reads that occur more than once. High levels of duplication may indicate some enrichment bias, like PCR over-amplification of a particular set of reads, or contamination. However, RNA-Seq libraries will normally show sets of reads at higher duplication bins. These potentially correspond to highly expressed transcripts which provide a high number of templates from the same region that are amplified and sequenced. For RNA-Seq analysis we do not remove these reads from the data.
Any peaks shown after ~ >1K may correspond to reads with high polyA content, mitochondrial RNA or and ribosomal RNA.
The profile is as expected for this type of experiment.




Trimming of low quality bases and adapters

Reads were trimmed using TrimGalore [2] v0.5.0. TrimGalore trims off low-quality base calls from the 3' end of the reads, before the adapter removal. Next, it finds and removes adapter sequences from the 3' end of reads. If reads after trimming are less than 20 bases long, they are discarded from the dataset. The percentage of reads that were removed after trimming, and the number of reads kept after trimming are shown in the table below. As with the per base sequence raw data plots, the per base sequence content should not vary too much across the length of the read. The adaptor trimming has indeed caused the expected drop in the proportion of A nucleotides at the end of the read, as mentioned above.

PolyA sequences may also affect the mapping of reads if the alignment tool used consists of alignment of reads from end-to-end. The tool used for mapping in this analysis is STAR, as explained in the following section. STAR does local alignment and clipping of 3'-end of reads in case such an alignment gives a better score. The number of reads remove due to the trimming process is very low as we would expect from good quality reads.
The number of reads removed from the data set is very low.

Differential gene expression

Differential Gene Expression Analysis was done using the counted reads and the R package edgeR [8] version 3.26.5 (R version 3.6.1), for the pairwise comparisons.

The pipeline used follows the next steps:

  1. Filter genes with low read counts. In the normal pipeline the number of samples to keep is selected so that at least half of the samples being tested have read counts of over 5 cpm (counts per million). If the groups have uneven numbers of replicates, e.g group A n=3 and group B n=6, the minimum number of samples is determined as the number of samples in the smallest group. In this case that would be 3. This is to ensure we capture genes that are upregulated in one group (on/off scenario).

  2. Calculate normalization factors. EdgeR computes effective library sizes using TMM (i.e trimmed mean of M-values) normalization. The normalization factors account for sequencing depth and RNA composition.

  3. Correct for GC content and gene length bias for all genes which pass filtering. GC content bias affects the amplification of fragments, which in turn affects read coverage and could confound true signals so therefore needs to be corrected for. We used the CQN bioconductor package (version 1.30.0) [9] which allows removal of a systematic GC content and gene length bias by a smoothing function. This should ensure that there is no bias towards GC rich reads, compared to those with low GC content. The same applies with gene lengths.

  4. Data exploration. MDS and scatter plots are produced to show sample relationships. MDS plots show distances in terms of BCV (i. e. biological coefficient of variation, which represents the square root of the common dispersion described in step 5 below). For both plots, data is filtered and normalised within each comparison group. Scatter plots use normalised and log-2 transformed cpm values.

  5. Estimate dispersion of each gene across samples and common dispersion. The common dispersion is the average dispersion across genes. A plot is shown for each comparison. Each genes average log-counts per million across samples (x-axis) is plotted against its coefficient of variation (BCV), so each point (referred to as tagwise in the plot legend) corresponds to a gene. The red line is the common dispersion which indicates the overall, or average, variability in the dataset. The blue line represents the trend of dispersion across all genes.

  6. Perform differential gene expression. We used the exact test edgeR approach to make pairwise comparisons between groups. We then adjust for multiple testing via the FDR (Benjamini-Hochberg) approach used by edgeR. Please refer to the edgeR documentation to know more about the statistical principles of the procedure [8].
    A table of the top 10 genes is shown for each comparison with the following information:
    • Ensembl ID
    • Gene name
    • logFC, which indicates the log (in base 2) difference between the two groups. The baseline group is always the first group in the comparison. The fold change is calculated by taking the average normalised count of the second group minus the average normalised count of the first (baseline) group. For example, if the comparison is A vs B then group A is the baseline group and the logFC is calculated using the following formula: Average(group B samples norm count) - average(group A samples norm count). Note that because we are using log values, this is equivalent to average(group B norm count) /average(group A norm count). A positive logFC value means higher expression in group B's samples (lower in group A's) and a negative logFC means lower expression in group's B samples (higher in group's A). A value of 1 is equivalent to a change of 2 fold, 2 is 4 fold, 3 is 8, etc. Using the value as a power of 2 gives you the real fold change
    • logCPM, which is the log-average abundance across samples.
    • P-value for the differential expression test.
    • FDR, which is the adjusted p-value for multiple testing.
    • The following columns show the normalised log CPM value for each gene in each sample. These have been appended to the model output results and are not the exact values used by the model to calculate the p-values and logFCs. EdgeR does not allow access to the values they use in the model, which have other factors added to each count for each sample, so the normalised log CPM values we add are the closest we can get to those values.

A link is provided for the full table of results for each comparison. The results are also plotted in MA plots, which show the average log-counts-per-million on the x-axis plotted against the logFC on the y-axis. Normally, blue lines are shown at logFC values of 1 and -1. It is important to consider the FDR together with the log 2 fold change (logFC) to determine a subset of interesting genes.

A note on the p-value: in some cases the adjusted p-value will have the same value for several genes, which might look like an error has occurred during the correction. This can happen as a results of how the FDR correction is performed. The correction steps are as follows:

1. Sort the p-values from low (rank 1) to high (rank N, with N the number of comparisons performed).   
2. Multiply the p-value by N.
3. Divide the p-value by its rank.
4. Check that the rank is kept.

The last point is the reason why adjusted p-values can share the same value across multiple genes; if the ranks are not kept and a higher ranked gene ends up with a lower adjusted p-value than a lower rank gene, the lower value of the higher ranked gene will also be used for the lower ranked p-value.
